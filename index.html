Node.js环境变量的配置（Windows)

说明：这里的环境配置主要配置的是npm安装的全局模块所在的路径，以及缓存cache的路径，之所以要配置，是因为以后在执行类似：npm install 模块名 [-g] 的安装语句时，会将安装的模块安装到【C:\Users\用户名\AppData\Roaming\npm】路径中，占C盘空间。
本文是将 nodejs 安装在 D:\Nodejs 目录下,以下操作可根据实际安装目录情况进行对应调整。

  1. 在安装目录下（如D:\Nodejs）新建两个空文件夹node_global（全局包存放目录）和node_cache（缓存目录）

  2. 打开命令行工具，如cmd，分别键入命令：

    • npm config set prefix “D:\Node.js\node_global”
    • npm config set cache “D:\Node.js\node_cache”

  3. 打开系统属性-高级系统设置-环境变量:

    • 在系统变量中新建变量名：NODE_PATH，变量值：D:\Node.js\node_global\node_modules
    • 编辑用户变量Path，将默认的C:\APPData\Roaming\npm修改为D:\Node.js\node_global

  4. 保存设置

  5. 可以使用官方推荐的cnpm命令行工具替代npm来加快下载速度：

    npm install -g cnpm –registry=https://registry.npm.taobao.org
